# knowledge-sequence — 顺序判断

## 核心判断

> **创业不是拼勇气，而是守顺序。顺序错了，越努力，越难回头。**

## 顺序框架

### 为什么顺序重要

创业是一连串高度耦合的动作。前面一步站错，后面很多动作看起来都在推进，其实是在放大那个错位。

> 你以为你在修正，实际上你只是在更快地朝错误方向跑。

### 最常见的顺序错误

| 错误顺序 | 正确顺序 |
|---------|---------|
| 产品开发 → 验证需求 | 验证需求 → 产品开发 |
| 招兵买马 → 确认方向 | 确认方向 → 招兵买马 |
| 执行推进 → 澄清判断 | 澄清判断 → 执行推进 |
| 组织扩张 → 生存结构走稳 | 生存结构走稳 → 组织扩张 |

### 顺序是生死线

顺序不是建议，是规律。你如果不尊重它，迟早要为此付代价。

> 顺序一反，方向就反；方向一反，跑得越快，死得越早。

## 判断能力

### 创业者的第一职责

不是执行，而是判断。不是先问"怎么把这件事做快一点、做漂亮一点"，而是先问：

- 这件事到底值不值得做？
- 现在该不该做？
- 先做哪一步，后做哪一步？
- 如果资源只能砸一处，应该砸在哪里？

### 判断先于执行

打工者习惯"先执行，再判断"。创业刚好相反，创业者必须"先判断，再执行"。

### 判断能力从哪里来

不是天赋，而是在一次次代价里，把对顺序的敬畏刻进骨子里。

> 这种稳，就是学习的结果。

## 关口识别

### 什么是关口

创业并非一团乱麻，而是由一道道"关口"组成的通关游戏。

两种困境：
1. **看不见门** — 被琐事淹没，不知核心问题在哪
2. **判断错误** — 虽然看到了问题，却用错了方法和资源

### 最容易被忽略的关口

| 关口 | 忽略的表现 |
|------|-----------|
| 客户关口 | 还没想清楚服务谁，就开始磨产品 |
| GTM关口 | 不知道从哪里切入，就开始做增长 |
| 模型关口 | 模型还没站稳，就开始扩张 |
| 人关口 | 环境还没建立，就开始招人 |
| 执行关口 | 判断还没固定，就开始盯执行 |

### 忙的真相

"忙"是创业初期最具迷惑性的烟雾弹。

> 这些忙碌，到底是在逼近结果，还是在逃避判断？

## 一语道破

| 问题 | 判断 |
|------|------|
| "我很忙但没结果" | "你不是在推进，是在用动作掩盖链条没搭好。" |
| "事情越做越多" | "你不是缺方法，是顺序全反了。" |
| "我先做起来再说" | "你是在更快地朝错误方向跑。" |

## 追问收尾

> 你现在，是不是正在用力做一件本不该现在做的事？
