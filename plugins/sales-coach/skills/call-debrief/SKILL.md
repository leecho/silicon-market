# 通话复盘与跟进

分析一次客户通话（录音、笔记或要点），生成结构化的教练反馈和完整的售后跟进包。

本技能优化目标：
- **基于证据的教练反馈**：具体到时间点的行为观察
- **可操作的跟进计划**：客户和销售双方的行动项
- **即用的跟进材料**：邮件草稿、CRM 更新文本、内部通报

## 适用场景

- 重要通话后的复盘和学习
- 为销售代表提供通话行为反馈
- 生成通话后的跟进邮件和行动项
- 团队内部分享通话要点和决策
- 对比同伴优秀实践提供改进建议

## 必要输入

- [必须] 通话证据：录音文字稿、通话笔记、会议纪要或详细要点
- [推荐] 客户/公司名称
- [推荐] 通话日期
- [可选] 参与者名单和角色
- [可选] 商机名称/阶段
- [可选] 同伴优秀通话片段（用于对比学习）
- [可选] 聚焦主题（定价/异议处理/发现/演示/谈判）

## 工作流程

### 第一部分：通话分析与教练反馈

#### 1. 提取关键时刻

对通话中每个关键时刻标注：

| 时间点 | 事件 | 标签 |
|--------|------|------|
| [时间] | [发生了什么] | 亮点 / 教练机会 / 关键决策 |

关注维度：
- 开场/议程控制
- 发现深度和排序
- 价值传递和证据
- 异议处理
- 干系人挖掘/采购流程
- 下一步承诺/MAP

#### 2. 生成教练反馈

```markdown
## 教练反馈

### 做得好的
- **[具体时刻]**：[为什么有效 + 可以扩大的原因]

### 教练机会
- **时刻**：[什么时候、买方说了/做了什么]
- **发生了什么**：[代表如何回应]
- **建议替代**：[具体的替代方法——精确到话术]
- **为什么重要**：[这会在交易中解锁什么]

### 技能关联
- **关联能力**：[在辅导计划中的哪个发展领域]
- **练习任务**：[下一通通话中应该尝试什么]
- **跟进方式**：[何时检查下次尝试的结果]
```

#### 3. 对比分析（可选，当提供同伴示例时）

```markdown
## 同伴最佳实践对比

### 可借鉴的模式
| 行为模式 | 同伴示例 | 为什么有效 | 如何应用 |
|---------|---------|-----------|---------|
| [模式] | [具体片段] | [效果] | [建议话术] |

### "偷师清单"
1. [具体行为 + 示例话术]
2. [具体行为 + 示例话术]
...
```

### 第二部分：跟进包

#### 4. 通话摘要

```markdown
## 通话摘要

- **客户**：[公司]
- **参与者**：[姓名/角色]
- **日期**：[日期]
- **类型**：[Discovery/Demo/谈判/Executive/续约]
- **核心发现**：[2-4 条要点]
- **决策/承诺**：[会上确认的事项]
- **风险/障碍**：[暴露的问题]
```

#### 5. 下一步行动

```markdown
## 下一步行动

### 客户侧
- [ ] [行动] — [负责人] — [截止日期/TBD]

### 我方侧
- [ ] [行动] — [负责人] — [截止日期/TBD]
```

#### 6. 跟进邮件草稿

```markdown
## 跟进邮件

**主题选项**：
1. [选项1]
2. [选项2]
3. [选项3]

**推荐主题**：[最佳选项]

**正文**：
> [150-220 字的专业跟进邮件，包含：
> - 感谢 + 回顾要点
> - 确认的下一步
> - 附件/资源承诺
> - 明确的行动号召]
```

#### 7. CRM 更新文本

```markdown
## CRM 更新（一句话）

[通话日期] — [下一步卖方行动] + [客户行动] + [预期结果]
```

#### 8. 内部通报草稿

```markdown
## 内部通报

**[会议标题] — [摘要]**

参与者：[列表]

**关键要点**：
- [要点1]
- [要点2]

**决策**：
- [决策1]

**行动项**：
- [ ] [行动] — @[负责人] — [截止]

**开放问题/风险**：
- [问题]

**来源**：[通话记录链接/笔记出处]
```

## 输出选项

用户可以请求完整包或部分输出：
- `完整复盘`（默认）：教练反馈 + 跟进包
- `仅教练`：只要教练反馈部分
- `仅跟进`：只要跟进包部分
- `仅邮件`：只要跟进邮件

## 规则

- **基于证据**：所有反馈基于通话摘录或可观察模式，不编造通话内容
- 不编造参与者、日期、承诺、定价或决策
- 缺失信息使用占位符（如 `<FirstName>`、`TBD`）
- 保持跟进邮件简洁专业，不包含敏感内部信息
- 教练反馈遵循"先问后说"原则——建议以问题形式引导时更有效
- CRM 更新严格一句话
- 不自动发送任何外部通信
