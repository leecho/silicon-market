# 商业案例构建

从客户上下文出发，构建客户导向的商业案例、ROI 叙事、价值模型和高管摘要。

**核心原则**：从客户出发，不是从产品出发。遵循这个链条：

`客户上下文 → 业务锚点 → 工作流 → 用例 → 价值驱动 → 指标 → 量化影响 → 叙事`

如果证据稀疏，产出清晰标注的结构性假设或跟进问题集，而非虚假的精确性。

## 适用场景

- 为特定客户构建采购商业案例
- 将发现笔记转化为价值叙事
- 为高管汇报准备 ROI 分析
- 构建 Champion 内部推动所需的价值材料

**不适用于**：没有客户角度的通用产品定位。

## 必要输入

- [必须] 客户/公司/细分市场/客户场景
- [必须] 工作流/计划/用例/决策/痛点/指标目标
- [可选] 买方角色、行业、方案范围、客户指标、当前工具、已知约束、证据材料、战略优先级、类比案例

## 工作流程

### 1. 确认客户和决策上下文

明确：
- 客户是谁
- 所在行业/细分市场
- 试图实现什么
- 什么压力使工作紧迫
- 哪个买方/团队/干系人重要
- 可见的约束是什么

**锚定在客户目标和业务压力上**，不要在这一步完成前就跳到产品推荐。

### 2. 识别业务工作流

在识别用例之前，理解客户试图改进的实际工作流/业务流程：

- 参与的角色和团队
- 主要步骤和交接
- 瓶颈、手工作业、延迟、质量问题
- 涉及的工具或系统
- KPI 和"好"是什么样的

**用业务语言命名工作流，不是产品语言。**

### 3. 优先排序用例

优先选择 1-3 个与客户目标和工作流约束匹配的用例：

- 解决什么工作流问题
- 为什么现在重要
- 哪个买方/团队关心
- 什么证据支撑
- 什么仍需验证

### 4. 映射价值驱动

使用以下价值类别：

| 价值类别 | 说明 |
|---------|------|
| `生产力提升` | 相同资源产出更多 |
| `成本降低` | 直接降低运营/人力/工具成本 |
| `风险降低` | 减少合规、安全、运营风险 |
| `收入加速` | 更快变现或提高转化 |
| `上市速度` | 缩短产品/功能交付周期 |

将每个用例映射到最少数量的价值类别。使因果链明确。

### 5. 识别所需指标

对每个用例，识别可信量化所需的最低指标集：

- 用户/工作流数量
- 任务量和频率
- 每任务时间和节省时间
- 人力成本和采纳率
- 周期时间、缺陷率、事故成本
- 转化率、收入影响、上市速度、工具支出

**不具备所需输入时不暗示可量化性。**

### 6. 量化（仅当输入支撑时）

当证据可信时，展示：
- 公式逻辑
- 已知输入
- 假设
- 低/基线/高场景
- 注意事项

**如果输入太弱无法量化**：保持案例为结构性，明确说明将案例从"结构性"推进到"财务可用"需要什么数据。

### 7. 起草客户导向叙事

将运营影响翻译为业务成果，用客户导向、销售可用的语言撰写：

- 客户优先级
- 客户挑战
- 相关工作流
- 相关用例
- 预期业务影响
- 为什么我方方案适合
- 什么假设重要
- 什么仍需验证

**语气应该是咨询式的、成果导向的、有据可依的、量身定制的——不是产品说明书。**

### 8. 输出默认包

```markdown
# 商业案例：[客户] — [日期]

## 高管摘要
[客户试图实现什么、为什么现在重要、最相关的工作流和用例、可能的价值故事、为什么我方方案相关]

## 战略计划
[客户的战略优先级和驱动力]

## 关键挑战
[当前面临的核心业务挑战]

## 优先工作流
[最重要的 1-3 个工作流及其痛点]

## 优先用例
[每个用例：解决什么、为什么现在、谁关心、证据、待验证]

## 价值假设
| 用例 | 价值类别 | 假设 | 影响估算 | 信心 |
|------|---------|------|---------|------|
| | | | | |

## 指标与假设
| 指标 | 已知值 | 假设值 | 来源 | 信心 |
|------|--------|--------|------|------|
| | | | | |

## ROI / 价值视图
[低/基线/高场景，仅当可量化时]

## 方案差异化
[将差异化与工作流、买方需求和预期效果关联]

## 证据与类比案例
[保持公开公司声明与客户本地证据分离]

## 注意事项与开放问题
[什么仍需验证、什么公开研究无法确定]
```

## 证据与假设规则

- 重要主张标注为 `已知`/`推断`/`假设`/`缺失`
- 公开来源的洞察与客户本地证据明确分离
- 类比案例可以塑造假设但不能假装是本客户的证据
- 如果信心低，说明原因，不将不确定性润色为精美文字

**绝对不要**：
- 编造客户数字
- 混淆客户事实和方向性假设
- 将公开战略语言转化为虚假 ROI
- 使用与客户价值脱节的通用营销语言
- 隐藏缺失数据

## 辅导视角

作为销售教练，在商业案例之后提供：
- 这个商业案例的"电梯测试"——30秒能讲清楚吗？
- Champion 内部推动时最可能面对的挑战
- 建议的下一步验证动作
