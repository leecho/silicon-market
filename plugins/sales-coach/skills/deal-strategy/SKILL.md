# 交易策略包

将交易证据转化为实用的策略包，帮助推进活跃交易。适用于已完成初步发现（Discovery）、有一定交易证据的场景。

## 适用场景

- 复杂交易的策略梳理和推进计划
- 交易卡住时的突破路径分析
- 重要交易的采购风险评估
- 买方委员会图谱和影响力分析
- 交易 review 前的准备

**不适用于**：首次发现通话（无历史背景）、通用开发探索、仅需议程的会议准备（用 `prepare-meeting`）。

## 必要输入

- [必须] 活跃客户/商机，或足够详细的交易笔记
- [必须] 足够的证据识别当前交易动态、障碍或下一步需求
- [可选] CRM 信息、通话记录、邮件往来、客户文档

**输出模式**：
- `完整策略包`（默认）
- `交易地图`
- `采购委员会`
- `采购风险`
- `行动计划`

## 工作流程

### 1. 确认交易锚点

- 明确客户和商机身份
- 确认当前阶段、金额、预期关单时间
- 识别交易动态（新建 / 续约 / 扩展 / 竞争置换）

### 2. 收集证据

从用户提供的信息和可用来源中收集：
- 交易基本事实（阶段、金额、时间线）
- 关键干系人及其立场
- 最近的互动和承诺
- 已知风险和障碍
- 竞品情况

### 3. 构建交易地图

```markdown
## 交易地图

### 业务计划与目标成果
[客户试图实现什么]

### 当前阶段与时间线
- **阶段**：[当前阶段]
- **预期关单**：[日期]
- **信心水平**：[高/中/低 + 原因]

### 活跃工作流
- [工作流1]
- [工作流2]
- [工作流3]

### 顶级障碍与依赖
- [障碍1 + 缓解计划]
- [障碍2 + 缓解计划]
```

### 4. 构建采购委员会图谱

为每个关键干系人记录：

| 人员 | 职位 | 部门 | 角色 | 立场 | 影响力 | 信心 | 证据来源 |
|------|------|------|------|------|--------|------|---------|
| | | | Economic Buyer / Decision Maker / Champion / Influencer / Blocker / Procurement / Legal / Security | 支持/中立/怀疑/阻碍 | 高/中/低 | 高/中/低 | |

**角色定义**：
- **Economic Buyer**：有预算权或最终业务决策权
- **Champion**：内部倡导者，希望方案成功并帮助导航组织
- **Blocker**：可能延缓或阻止采购的人
- **Influencer**：能影响决策但无最终决定权

### 5. 构建风险登记册

#### 风险类型

| 类型 | 说明 |
|------|------|
| `procurement_process` | 采购流程风险 |
| `legal_terms` | 法律条款风险 |
| `security_review` | 安全审查风险 |
| `privacy_or_data_governance` | 隐私/数据治理风险 |
| `budget_or_funding` | 预算/资金风险 |
| `executive_alignment` | 高管对齐风险 |
| `stakeholder_coverage` | 干系人覆盖风险 |
| `timeline_dependency` | 时间线依赖风险 |
| `technical_dependency` | 技术依赖风险 |

#### 严重度评估

- **高**：可能阻止签约或导致实质性关单日期延迟
- **中**：可能延缓势头或需要升级，但尚未明确阻断交易
- **低**：可管理的摩擦，有明确负责人和积极缓解措施

#### 风险登记表

| ID | 风险摘要 | 类型 | 严重度 | 可能性 | 信心 | 负责人 | 缓解措施 | 目标日期 | 来源 |
|----|---------|------|--------|--------|------|--------|---------|---------|------|
| R1 | | | | | | | | | |

#### 精度规则
- 不使用精确日历日期，除非有直接证据支撑
- 低信心或证据稀疏时，使用相对时间（本周/下周/本月）或 TBD
- 客户侧负责人未确认时标注为"建议负责人"

### 6. 生成优先行动计划

每个行动必须包含：

```markdown
## 优先行动（5-7 个高信号行动）

1. **行动**：[具体行动]
   - **负责人**：[姓名]（或"建议负责人：[姓名]"）
   - **截止**：[日期 或 TBD]
   - **预期结果**：[行动达成后的效果]
   - **关联风险**：[R1, R2...]
   - **关联干系人**：[姓名]
```

**优先级排序**：
1. 高严重度 + 高可能性的风险缓解
2. 近期截止的中等风险
3. 阻碍推进的干系人覆盖缺口
4. 其他行动

### 7. 标注证据缺口

```markdown
## 证据缺口

| 缺口 | 影响 | 最小收集步骤 |
|------|------|-------------|
| [缺失信息] | [对策略的影响] | [获取建议] |
```

### 8. 分离事实与推断

所有推断而非直接陈述的结论，前缀标注 `推断：`

## 辅导视角

作为销售教练，在策略包之后提供：
- MEDDPICC 完整性检查（哪些关键要素缺失）
- 一个苏格拉底式问题帮助销售思考交易关键盲点
- 与类似交易模式的对比观察

## 规则

- 不编造干系人、审批、障碍、负责人、承诺、金额或日期
- 事实与推断严格分离
- 证据冲突时降低信心并标注证据缺口
- 证据稀疏时仍可输出，但明确标注限制
- 保持只读——除非用户明确要求，不建议写入外部系统
