# 会议准备简报

为销售人员或客户经理准备即将到来的客户会议。聚焦于当前会议本身：议程、决策点、高优先级话题、可能的障碍和下一步行动。将更广泛的客户/产品/交易/支持背景保持简短且从属。

## 适用场景

- 客户面谈或拜访前的准备
- 重要商务会议前的情报收集
- 日常最重要会议的快速准备
- 续约/扩展/谈判会议的策略准备

**不适用于**：常规内部团队会议、1:1、面试、绩效对话。仅在有明确商业利害关系时才为非客户会议准备。

## 必要输入

- [必须] 公司/客户名称或会议标题
- [必须] 会议参与者或参与者所在组织
- [必须] 会议目标、议程、话题或会议类型
- [可选] 过往上下文：通话记录、笔记、邮件、文档、CRM 信息

**如输入不完整**：先通过 Web 搜索获取公司公开信息以丰富背景，再向用户确认缺失的关键信息。

## 工作流程

### 1. 确认会议锚点

- 明确客户公司、关键参与者、会议时间和目标
- 如信息模糊，列出最多 5 个候选项供用户确认

### 2. 收集背景信息

信息来源优先级：
1. 用户直接提供的笔记、文档、上下文
2. 工作区内相关文件（如有）
3. Web 搜索获取公司公开信息（财报、新闻、产品页面）

收集维度：
- **客户公司**：行业、规模、近期动态、战略重点
- **参与者**：职位、职能、决策角色、与我方过往互动
- **交易背景**：当前阶段、金额、时间线、竞品情况
- **历史互动**：上次会议要点、承诺事项、未决问题

### 3. 构建会前简报

#### 输出格式

```markdown
# 会议准备简报：[客户] — [日期]

## 会议概要
- **客户**：[公司名]
- **参与者**：[姓名/职位列表]
- **时间**：[日期时间]
- **目标**：[本次会议的核心目标]
- **类型**：[Discovery / Demo / 谈判 / Executive / 续约 / 其他]

## 背景上下文
[简明扼要的客户和交易背景，仅包含影响会议策略的信息]

## 议程与话题优先级
1. [最高优先级话题] — 为什么重要 + 准备要点
2. [次优先级话题] — 为什么重要 + 准备要点
3. [其他话题]

## 预期问题与异议
| 可能的问题/异议 | 建议应对 |
|----------------|---------|
| [问题1] | [应对策略] |
| [问题2] | [应对策略] |

## 关键决策点
- [需要在会议中推动的决策]
- [需要确认的信息]

## 风险与障碍
- [可能阻碍进展的因素及应对]

## 行动计划
- **会前**：[需要提前完成的准备]
- **会中**：[关键动作和提问]
- **会后**：[预期的下一步]

## 信息缺口
- [尚未确认的关键信息及获取建议]
```

### 4. 提供辅导视角

以销售教练的身份，额外提供：
- 本次会议最大的教练观察点
- 一个具体的行为建议（如提问技巧、节奏控制）
- 与 MEDDPICC 框架的关联检查

## 规则

- 不编造参与者、承诺、日期或交易细节
- 所有事实必须有来源支撑或明确标注为推断
- 保持简报简洁聚焦，不超过 A4 两页篇幅
- 优先提供可操作的信息，而非泛泛的背景
- 用客户的语言而非产品术语描述业务场景
