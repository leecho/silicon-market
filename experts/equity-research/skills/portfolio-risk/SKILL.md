# 投资组合风险管理（Portfolio Risk）

## 功能说明

将投资论点和交易建议转化为完整的风险管理方案。核心目标：确定合适的仓位大小、设计适当的对冲（如需要）、明确保留的风险敞口，以及制定持续的监控和调整规则。

## 核心原则

1. **仓位大小由最紧约束决定**：流动性、波动率、集中度、止损预算等约束中，取最严的那个
2. **对冲和仓位不独立**：对冲改变了有效敞口，须综合考虑
3. **规则先于情绪**：在冷静时制定规则，在波动时执行规则
4. **Alpha vs 非预期风险**：识别你有意承担的风险（alpha 来源）和无意承担的风险（需要对冲/规避）

## 工作流

### Phase 1：投资论点确认

1. 确认底层投资论点和方向（来自 long-short-pitch 或其他分析）
2. 明确预期持有期限
3. 确认信心水平和证据完备度
4. 识别 alpha 来源：你赚钱是因为什么风险暴露？

### Phase 2：仓位大小（Position Sizing）

#### 约束检查清单

| 约束类型 | 具体限制 | 隐含最大仓位 |
|---------|---------|------------|
| 止损预算 | 单笔最大亏损 ≤ 组合 X% | 仓位 = 预算 / 预期最大跌幅 |
| 波动率 | 单仓位贡献波动 ≤ X% | 根据个股波动率反推 |
| 流动性 | 退出时间 ≤ X 天 | 仓位 ≤ X天 × 日均成交 × 参与率 |
| 集中度 | 单仓位 ≤ 组合 X% | 硬上限 |
| 相关性 | 同类敞口合计 ≤ X% | 考虑已有相关仓位 |

#### 最终仓位 = min(各约束隐含上限)

### Phase 3：对冲评估（如适用）

1. **需要对冲什么**：
   - Beta / 系统性风险
   - 行业 / 因子风险
   - 事件尾部风险
   - 外汇风险
2. **对冲工具选择**：
   - 行业 ETF 做空
   - 指数期货
   - 期权保护（Put 买入 / Collar）
   - 配对交易（做空同行业弱势标的）
3. **对冲成本与收益**：
   - 对冲成本 / Carry
   - 保护程度
   - 对 Alpha 的影响（是否过度对冲丢失收益）
4. **不对冲的替代方案**：
   - 缩小仓位达到同等风险
   - 成本对比

### Phase 4：风险暴露总结

```
有意保留的敞口（Alpha来源）:
• [描述你有意承担的风险]

已对冲/规避的敞口:
• [描述已对冲的风险]

残余/未对冲的敞口:
• [描述仍然承担但未对冲的风险及原因]
```

### Phase 5：监控规则

制定明确的"如果X发生，则执行Y"规则：

| 触发条件 | 行动 | 优先级 |
|---------|------|-------|
| 股价跌破 ¥XX（止损） | 减仓至 X% 或清仓 | 强制 |
| 论点证伪条件触发 | 3 日内清仓 | 强制 |
| 目标价达到 | 减仓至 X% | 纪律 |
| 波动率突然放大 >X% | 审视仓位是否过大 | 审视 |
| 流动性恶化 | 提前启动退出 | 审视 |
| 催化剂落地且正面 | 可加仓至 X% | 机会 |
| 估值重新达到吸引力 | 可回补至 X% | 机会 |
| 对冲到期/失效 | 评估续作或裸持 | 纪律 |

### Phase 6：定期复盘计划

- 每 [周/月] 检查一次论点是否仍然成立
- 业绩发布后 24 小时内更新评估
- 仓位超过 X% 时增加监控频率

## 注意事项

- 仓位大小是风险管理的第一道防线，比对冲更重要
- "不对冲 + 小仓位"往往优于"大仓位 + 昂贵对冲"
- 对冲可能引入新的风险（基差风险、保证金风险）
- 监控规则必须在建仓前制定，不是事后再想
- 止损是纪律问题，不是预测问题——不需要论证"会不会跌到止损"
- 对冲不能消除所有风险，只能改变风险的形态和成本
- 流动性是非线性的：正常时期的流动性不代表压力时期
- 本工具输出研究参考，不构成个人投资建议
