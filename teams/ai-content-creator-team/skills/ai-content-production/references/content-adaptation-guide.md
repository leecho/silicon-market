# 素材改编完整指南

本文档涵盖三大素材改编能力：视频翻译、视频分析和多媒体合成。

## WorkBuddy 可用工具

| 工具/模型 | 调用方式 | 能力 |
|-----------|----------|------|
| **YT-VITA** | `youtu-vita`，通过多模态内容生成 Skill 调用 | 视频/图片内容理解分析，视频结构解析、图像目标检测 |
| **多模态内容生成** Skill | 说出生成意图自动触发 | 辅助合成任务 |

---

## 一、视频翻译（跨语言本地化）

### 1.1 核心技术栈

| 技术 | 功能 | 效果 |
|------|------|------|
| AI 声纹克隆 | 保留原始说话者的声音特征 | 翻译后听起来仍是原作者在说目标语言 |
| 情感保留 | 分析并复制原始音频的情感曲线 | 兴奋/平静/悲伤等情绪在翻译后保持一致 |
| 唇型同步 | 调整视频中人物唇部动作 | 口型与翻译后语音自然匹配 |
| 字幕翻译 | 识别并翻译硬编码字幕 | 画面上的文字同步翻译 |

### 1.2 支持语言

**完整支持的语言对**：
- 中文（普通话）↔ 英文
- 中文 ↔ 日文
- 中文 ↔ 韩文
- 中文 ↔ 西班牙文
- 英文 ↔ 日文
- 英文 ↔ 韩文
- 英文 ↔ 西班牙文
- 更多语言组合持续扩展中

### 1.3 最佳效果条件

| 条件 | 推荐 | 原因 |
|------|------|------|
| 音频清晰度 | 高清音频，低背景噪音 | 声纹克隆依赖清晰的原始音色 |
| 说话人数 | 单人为佳 | 多人场景声纹区分困难 |
| 语速 | 适中（120-180 字/分钟） | 过快影响唇型同步精度 |
| 视频分辨率 | ≥720p | 唇型检测需要足够的面部像素 |
| 面部可见度 | 正面或3/4侧面 | 侧面/遮挡降低唇型同步效果 |

### 1.4 翻译流程

```
Step 1: 获取源视频 URL
Step 2: 提交翻译任务
  - 指定目标语言
  - 描述特殊要求（如"保留背景音乐"、"仅翻译字幕"）
Step 3: 等待处理
  - 首次检查：210 秒后
  - 后续检查：每 30 秒
  - 超时：20 分钟
Step 4: 获取翻译后视频 URL
```

### 1.5 翻译请求示例

**中文→英文**：
```
将这段中文产品介绍视频翻译为英文，要求：
- 保留原始说话者的声音特征
- 保持激动和热情的语调
- 翻译视频中的字幕文字
- 背景音乐保持不变
```

**英文→日文**：
```
Translate this English tutorial video to Japanese with natural voice cloning and lip sync. Maintain the instructional tone and keep background music unchanged.
```

### 1.6 质量影响因素

| 因素 | 对声纹克隆的影响 | 对唇型同步的影响 |
|------|----------------|----------------|
| 背景噪音 | 高（降低克隆精度） | 低 |
| 多人对话 | 高（混淆音色） | 中（多张嘴） |
| 快速语速 | 中（音素压缩） | 高（口型变化快） |
| 方言/口音 | 中（非标准发音） | 低 |
| 面部遮挡 | 无影响 | 高（无法检测嘴部） |

## 二、视频内容分析（基于 YT-VITA）

> 视频内容分析使用 WorkBuddy 内置的 **YT-VITA** 模型（调用参数 `youtu-vita`），支持视频/图片内容理解分析、视频结构解析、图像目标检测。通过 `多模态内容生成` Skill 自动触发。

### 2.1 分析维度

#### 视觉分析
- **场景检测**：识别视频中的场景切换点和场景类型（室内/室外/特写等）
- **物体识别**：检测画面中的关键物体（人物、产品、道具等）
- **动作识别**：理解画面中人物/物体的动作和行为

#### 音频分析
- **语音识别**：转录对话内容为文字
- **音乐检测**：识别背景音乐的存在和类型
- **音效识别**：检测特殊音效和环境声

#### 语言理解
- **对话分析**：理解对话的主题、意图和情感
- **情感检测**：分析整体情感走向（正面/负面/中性）

#### 多模态融合
- 综合视觉+音频+语言，生成全面的内容理解
- 识别各模态间的关联性（如悲伤配乐+哭泣画面=悲伤场景）

### 2.2 输出类型

#### 类型 A：精彩片段提取（Highlight Extraction）

**输出格式**：
```
片段列表:
1. [00:23-00:35] 精彩时刻描述 → clip_url_1
2. [01:12-01:28] 精彩时刻描述 → clip_url_2
3. [03:45-04:02] 精彩时刻描述 → clip_url_3
```

**识别逻辑**：
- 高能量时刻（音量峰值+快速运动）
- 情感高潮（惊喜/笑声/感动）
- 视觉冲击（特效/转场/精彩镜头）
- 观众反应（弹幕/评论密集时段）

**适用场景**：
- 从长视频中提取高光合集
- 制作预告片
- 筛选适合社媒传播的片段
- 创建视频摘要

#### 类型 B：视频理解（Video Understanding）

**输出格式**：
```markdown
## 视频摘要
[2-3句话概括视频核心内容]

## 关键词
tag1, tag2, tag3, tag4, tag5

## 剧情时间线
- 00:00-00:30: [开场/引入]
- 00:30-02:00: [主要内容1]
- 02:00-03:30: [主要内容2]
- 03:30-04:00: [结尾/总结]

## 情感曲线
开头(平静) → 中段(兴奋) → 高潮(激动) → 结尾(满足)
```

### 2.3 处理限制

| 限制项 | 数值 | 说明 |
|--------|------|------|
| 最大视频时长 | 2 小时 | 超过此长度需要分段处理 |
| 最大等待时间 | 20 分钟 | 超时则报告失败 |
| 处理速度 | ~10分钟/1分钟视频 | 仅为估算，实际因内容复杂度而异 |
| 首次检查 | 210 秒 | 提交后首次轮询间隔 |
| 后续检查 | 30 秒 | 后续轮询间隔 |

### 2.4 分析请求示例

**精彩片段提取**：
```
分析这段 30 分钟的 Vlog 视频，提取 5 个最精彩的片段，每个片段 15-30 秒，适合在短视频平台传播。
```

**视频理解**：
```
对这段产品发布会视频进行全面分析，输出摘要、关键词和时间线。
```

**制作预告片**：
```
从这段 1 小时的纪录片中，提取最具视觉冲击力的片段，用于制作 30 秒预告片。
```

## 三、多媒体合成

### 3.1 合成能力矩阵

| 合成类型 | 输入 | 输出 | 典型用途 |
|---------|------|------|---------|
| 多视频拼接 | 多个视频 URL | 单一合并视频 | 素材汇编、花絮集 |
| 图片轮播视频 | 多张图片 URL | 幻灯片视频 | 产品展示、相册 |
| 视频+背景音乐 | 视频+音频 URL | 带配乐视频 | Vlog、宣传片 |
| 视频蒙太奇 | 多个视频/图片 | 动态混剪 | 宣传片、MV |

### 3.2 支持格式

| 类型 | 支持格式 |
|------|---------|
| 视频 | MP4, MOV |
| 图片 | JPG, PNG |
| 音频 | MP3, WAV |

### 3.3 合成流程

```
Step 1: 收集所有媒体素材 URL（视频/图片/音频）
Step 2: 用自然语言描述合成需求
Step 3: 提交合成任务（素材作为 multiInputs 数组传入）
Step 4: 等待处理
  - 首次检查：210 秒后
  - 后续检查：每 30 秒
  - 超时：20 分钟
Step 5: 获取合成视频 URL
```

### 3.4 合成请求示例

**图片轮播视频**：
```
将这 5 张产品图片制作为视频，每张显示 3 秒，使用淡入淡出转场效果。
multiInputs: [photo1.jpg, photo2.jpg, photo3.jpg, photo4.jpg, photo5.jpg]
```

**视频+背景音乐**：
```
将这 3 段旅行视频合并为一个，并添加这首轻松的背景音乐。
multiInputs: [clip1.mp4, clip2.mp4, clip3.mp4, background_music.mp3]
```

**视频蒙太奇**：
```
用这些素材创建一个动态的视频蒙太奇，带有节奏感强的转场。
multiInputs: [vacation1.mp4, vacation2.mp4, vacation3.mp4]
```

### 3.5 处理时间

- **预估**：约 10 分钟生成 1 分钟输出视频
- **影响因素**：输入素材数量、输出时长、转场复杂度
- **超时**：20 分钟

### 3.6 注意事项

1. **输入顺序**：multiInputs 数组的顺序决定最终视频的播放顺序
2. **格式混用**：同一次合成可以混合视频、图片和音频
3. **音频处理**：当同时存在视频原声和背景音乐时，需在 prompt 中说明混音比例
4. **分辨率**：输出分辨率由系统根据输入素材自动决定
5. **转场选择**：可在 prompt 中指定转场类型，或让系统自动选择

## 四、任务优先级

当需要同时处理多种改编任务时，按以下优先级排序：

1. **视频翻译**（最高）：通常时间敏感，客户等待
2. **精彩片段提取**：为后续剪辑提供素材
3. **视频理解分析**：为内容策略提供依据
4. **多媒体合成**：通常可安排在其他任务之后

---

## 五、通用注意事项

1. **等待时间管理**：所有任务首次检查 210 秒，后续 30 秒间隔，最大 20 分钟
2. **结果临时性**：输出的媒体 URL 可能有有效期，建议及时保存到内容库
3. **内容安全**：不处理违规内容，不对包含未成年人的视频做不当改编
4. **大文件注意**：超长视频处理时间更长，建议先裁剪无关部分
5. **质量预期**：翻译质量受源视频质量影响，噪音大/多人说话时效果下降
6. **语言匹配**：所有输出语言与用户请求语言一致
7. **错误处理**：超时或失败时，提供清晰的错误说明和重试建议
