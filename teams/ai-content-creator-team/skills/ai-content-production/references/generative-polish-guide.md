# 发布级视觉精修指南（Generative Polish）

## 概述

Generative Polish（生成式精修）是一种有边界的创意技术：用确定性工具构建信息关键层（文字、数据、Logo），用 AI 图像生成创建视觉精修层（背景、纹理、光效），然后确定性地合成最终资产。

目标：获得高端视觉品质，同时不让 AI 生成工具篡改已审核的文案、扭曲图表数据、修改产品声明、虚构 UI 或违反社媒投放规范。

---

## 一、核心原则

### 分层所有权

| 层 | 所有者 | 原因 |
|----|--------|------|
| 文案、声明、标签、CTA | 确定性工具（文本/SVG/HTML） | 文字必须精确可审核 |
| 数据、图表、坐标轴、数字 | 确定性工具（图表库/SVG） | 数值必须准确无误 |
| Logo 位置、安全区域、尺寸 | 确定性工具（模板/SVG） | 品牌规范必须确定性执行 |
| 文件命名、清单、预览包 | 确定性工具 | 交付必须可重复 |
| 背景氛围、光效、纹理、景深 | AI 图像生成 | 这些受益于生成式视觉品质 |
| 编辑场景、抽象隐喻、Hero视觉 | AI 图像生成 | 高精修但通常非声明关键 |

**关键规则**：不要让 AI 图像生成拥有精度关键层，除非用户明确接受风险且产出仅为探索性质。

---

## 二、默认工作流

### 步骤 1：定义资产体系

- 确定目标格式、宽高比、渠道、审核状态
- 判断工作是探索性还是发布级

### 步骤 2：锁定确定性基底

- 创建精确的文字布局、CTA、Logo区域、数据图表
- 使用可编辑变量保持文字和数据的可修改性
- 工具选择：Markdown/HTML/CSS/SVG/Python 图表库

### 步骤 3：生成精修层

使用 AI 图像生成创建背景/纹理/光效：

**标准 Prompt 模板**：
```
创建一个高品质的视觉背景层，用于[具体用途，如"科技产品发布社媒海报"]。
要求：
- 禁止包含任何文字、Logo、UI元素、图表、数字、产品声明、水印
- 保留干净的负空间区域，位于[画面上方/中央/下方]，用于后续叠加确定性内容
- 使用[具体视觉风格]风格的光效、景深、纹理和构图
- 色调以[主色调]为主，营造[情绪]氛围
- 适合[目标渠道]的[宽高比]格式
```

**引用已有设计作为参考时**：
```
仅将提供的图片作为构图/风格参考。
禁止重写、添加或解读任何文字。
禁止修改图表数值、坐标轴、Logo、产品UI或来源标注。
仅关注背景精修：光效、景深、纹理和氛围提升。
```

### 步骤 4：确定性合成

- 将 AI 生成层作为底层
- 确定性地叠加文字/Logo/图表/CTA
- 导出最终固定尺寸资产

### 步骤 5：验证与交付

检查清单：
- [ ] 尺寸正确（目标渠道规范）
- [ ] 文字精确无误（未被 AI 修改）
- [ ] 图表数据准确
- [ ] Logo 在安全区域内
- [ ] 对比度/可读性符合标准
- [ ] 文件命名规范

## 三、常见模式

### 社媒轮播图（Social Carousel）

```
确定性层：卡片网格、标题、图表、CTA、来源标注
生成层：  卡片背景纹理、编辑场景、隐喻性Hero图
合成方式：确定性工具导出最终固定尺寸 PNG
```

### 数据卡片（Chart Card）

```
确定性层：图表（含标签、坐标轴、数据点）、标题、来源
生成层：  精修背景/材质处理
合成方式：图表叠加在生成背景之上
```

### 产品发布 Hero 图（Launch Hero）

```
确定性层：发布名称、CTA、产品链接、审核标签
生成层：  Hero场景/抽象视觉/氛围处理
合成方式：确定性布局 + AI Hero 背景
```

**注意**：Hero 图如需探索多方向，应先进行"广告方向策划"确定方向后再精修。避免将精修当做方向探索工具。

### 社媒适配包（Format Pack）

```
源：  确定性全尺寸主图
流程：从主图导出各平台尺寸变体
规则：适配过程不得改变文案、图表、Logo或声明
```

## 四、护栏规则

### 绝对禁止

- ❌ 不用 AI 生成最终法律声明、价格信息、可用性声明、安全评级
- ❌ 不依赖 AI 渲染的文字/数字/代码/图表/表格/Logo 作为发布版本
- ❌ 不创建虚假产品 UI、非授权品牌标志、用户证言
- ❌ 不隐瞒图像层为 AI 生成（需标注时必须声明来源）
- ❌ 不让视觉精修覆盖可读性、对比度和无障碍标准

### 如果 AI 生成中出现文字/图表

如果 AI 图像输出中意外包含文字或图表元素，视为**非最终版本**——必须用确定性工具重建这些元素后再合成。

---

## 五、适用场景判断

| 场景 | 是否适合精修 | 说明 |
|------|------------|------|
| 探索多个广告方向 | ❌ 不适合 | 先用广告策划确定方向 |
| 已选定方向，需发布级品质 | ✅ 适合 | 精修流程 |
| 快速原型/草图 | ❌ 不需要 | 直接 AI 生成即可 |
| 有精确数据/文字的正式发布物 | ✅ 适合 | 分层精修 |
| 社交媒体日常内容 | ⚠️ 视情况 | 简单内容直接生成；正式活动用精修 |

## 六、与团队协作

### 精修流程中的角色分工

| 步骤 | 负责人 | 产出 |
|------|--------|------|
| 定义资产体系 | 创意总监/策划师 | 需求文档 |
| 文案/CTA | copywriter | 精确文本 |
| 生成视觉层 | image-creator | AI生成的背景/纹理 |
| 确定性合成 | image-creator | 最终合成资产 |
| 尺寸适配 | image-creator | 多平台导出 |
| 质量审核 | 创意总监 | 审核通过/修改意见 |

### 精修触发条件

主理人在以下情况下应调度精修流程：
1. 用户明确要求"发布级品质"或"正式版"
2. 内容包含精确数据/品牌标志需要保护
3. 需要多平台尺寸适配的正式物料
4. 用户已选定创意方向，进入最终产出阶段
